# 🛰️ Safety System & Fault Tollerance

The TIG Rotator Controller implements a multi-layer safety architecture to protect both the operator and the mechanical hardware in industrial welding environments.

## 1. Safety State Machine
The system transitions through a rigorous state machine (defined in `control.h`):
- **STATE_IDLE:** Default state. Motor `ENA` is `HIGH` (Disabled).
- **STATE_RUNNING:** Motor is active. Acceleration ramps are active.
- **STATE_STOPPING:** Triggered by UI stop. Motor decelerates smoothly.
- **STATE_ESTOP:** **CRITICAL FAULT.** Triggered by GPIO 33 interrupt. All motion halted instantly.

## 2. Emergency Stop (E-STOP)
- **Hardware Interrupt:** GPIO 33 is configured with a falling-edge interrupt.
- **Normally Closed (NC) Logic:** The button pulls the pin `LOW`. If the wire is cut or the button is pressed, the pin goes `HIGH`, triggering the interrupt in microseconds.
- **Software Halt:** The ISR (Interrupt Service Routine) calls `stepper->forceStop()` and sets `PIN_ENA` to `HIGH` (Disconnected) immediately.

## 3. Acceleration & Deceleration (Anti-Stall)
To prevent step loss and mechanical shock:
- **Linear Acceleration:** Ramps are generated by the hardware-timer-based `FastAccelStepper` engine.
- **Default Acceleration:** `5000 steps/s²` (Configurable via `ACCELERATION` in `config.h`).
- **Soft Stop:** Standard stop commands trigger a deceleration ramp rather than an instant halt.

## 4. Hardware Watchdog (TWDT)
The ESP32-P4's internal **Task Watchdog Timer** is assigned to:
- **Motor Task:** Ensures pulses are being generated.
- **Control Task:** Ensures the state machine is responsive.
- **Safety Task:** Ensures the E-STOP logic is alive.
If any task freezes, the system triggers a hardware reset, immediately disabling the motor driver via the default pull-up state of the `ENA` pin.
